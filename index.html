Angular JS Auth Seed

This is a modified version of the AngularJS seed project that includes a login / authentication system. It includes 3 basic demo pages (create account, login, protected content), as well as basic PHP server-side code for the app.

On default, when a page loads, the app asks the server if the user is logged in, so that any non-login or create page will be protected. You can also configure the system to only require a login when the app is rejected sensitive content due the the user not being logged in.

How It Works

The system works by intercepting 401 Not Authorized responses from the server (in for example a $http query). Any 401 Not Authorized response will trigger a rerender of the login screen. The page the user attempted to access when given the 401 response is saved, and returned to once authentication is attained again.

Getting Started

This is used like the normal AngularJS Seed. Clone it into a directory and host it with a PHP/Mysql server. Use the .sql file to create the basic database structure. Configure the system for your database in conn_mysql.php

Requirements

PHP for default server-side

MySql for default database
